    맥 키보드 한/영 키 설정

    2020/01/25
    기록
    Mac, Karabiner

    Karabiner 설치

    여기서는 Brew로 맥의 GUI 프로그램을 설치하는 brew cask방법으로 설치한다.

    $ brew cask install karabiner-elements

    Karabiner 구성파일 셋업

    출처: 작은곰의 작업실

    1. ⌘ command + ⇧ shift + .을 누르면 숨겨진 파일을 보여준다.
    2. ~/.config/karabiner/assets/complex_modifications 의 경로를 찾아 이동한다.
    3. 혹은 ⌘ command + ⇧ Shift + g 실행해서 위의 경로를 입력해서 이동한다.
    4. 애플키보드 한영전환/한자키 지정 (rev.1.13d) 다른 이름(KorEng-Hanja.json)으로 저장해서 위의 디렉터리로 파일 복사 혹은 이동한다.

    Karabiner-Elements 실행

    1. 3번째 Complex modifications 탭을 클릭한다.
    2. 아래 Add rule 클릭하고, Enable All 클릭해서 등록한다.
    3. 아래처럼 나오는지 확인하고 종료한다.
    4. 오른쪽의 ⌘ command 키가 잘 동작하는지 눌러보자.
    5. 패러럴즈의 윈도우에서도 되는지 확인한다.

    맥과 패러럴즈(윈도우) 키보드 싱크 회피

    위와 같이 했다면 아래 그림과 같이 A과 한 보여야 한다.

    하지만, 맥과 패러럴즈(윈도우10) 전환하면서 사용하다 보면, 또는 패러럴즈(윈도우10)을 장시간 접근하지 않고 맥에 있다가 패러럴즈(윈도우10)로 전환하다 보면, 위처럼 보이는 것도 A과 한 사라지거나 하나만 보인다.

    따라서,

    윈도우 10 이하 버전에서 발생하는 것인지 모르겠으나, 윈도우10에서 키보드가 아래 그림처럼 ENG 형태가 보인다.

    패러럴즈는 OS를 설치하면서 키보드의 매핑을 기본 값으로 설정하고 있다고 한다.

    윈도우를 사용하고 있다 맥으로 전환되거나 혹은 윈도우 리부팅을 해도 ENG형태가 보이는 현상이 즉시 나타나지 않아도 시간이 지나고 사용하면서 발생하게 되었다.

    이것은 이것은 맥의 키보드와 패러럴즈 가상 형태의 키보드가 Sync 되어 있기 때문이라는데, 이것을 해결 하기 위해서는 다음과 같은 가상화 설정 변경으로 해결 할 수 있다. 일주일 째 사용하고 있는데 문제 없이 잘 사용하고 있어 기록으로 남긴다.

    1. 가상머신 Windows 10이 실행 중이면, Suspend가 아닌 완전히 OS 종료를 한다.
    2. 패러럴즈를 종료한다.
    3. 그림처럼 ⌘ command + ⇧ Shift + g 실행하여 /Users/{User 이름}/Parallels/로 이동한다.
    4. {가상 머신 이름}.pvm 형태의 파일의 가상머신이 보일 것이다(여기서 가상머신 이름은 Windows 10.pvm). 마우스 우클릭 선택하여, 이 가상머신의 컨텍스트 메뉴의 패키지 보기를 한다.

    5. 텍스트 편집기로 config.pvs 파일을 오픈한다.
    6. KeyboardLayoutSync를 찾는다. 바로 아래의 Enabled의 값이 1로 되어 있을텐데 0으로 변경한다.
    7. 패러럴즈 시작하고, 가상머신을 시작한다.

    누군가에게 도움이 되었기를 …

    하지만, 맥과 윈도우 전환하다 보면 여전히 작동하지 않는다.
